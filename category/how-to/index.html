    docker 基础

    基本概念

    • 镜像是将一些工具环境的打包
    • 容器是镜像的实例化

    Docker 仓库

    • Docker hub

    Dockerfile

    Dockerfile 可以用来创建镜像，文件名叫做 Dockerfile。

    DockerFile 关键指令

    • FROM：制定基础镜像，所有镜像都基于一个基础镜像
    • RUN：在镜像构建的过程中执行命令（如安装依赖）
    • COPY/ADD：将文件从主机复制到镜像中
    • WORKDIR：设置容器内的工作目录
    • ENV：设置环境变量
    • EXPOSE：声明容器开放的端口
    • CMD/ENTRYPOINT：指定容器启动时的默认命令

    Docker Compose

    可以管理多个容器环境。

    docker 安装 sqlserver 简易版

    TLDR

    使用下面的命令

    sqlcmd create mssql --accept-eula --name mssql

    安装

    首先需要 docker 已经安装，现在先安装 sqlcmd

    winget install sqlcmd

    sqlcmd 可以直接创建 sqlserver 容器，查看可选项

    sqlcmd create mssql --help
# 查看可以安装的版本
sqlcmd create mssql get-tags

    安装 sqlserver 2022 并且创建实例，然后还原 Wide World Importers 实例，可以不还原。

    sqlcmd create mssql --accept-eula --tag 2022-latest --using https://github.com/Microsoft/sql-server-samples/releases/download/wide-world-importers-v1.0/WideWorldImporters-Full.bak

    我不想放数据，就执行了

    sqlcmd create mssql --accept-eula --name mssql --hostname dog --port 1433

    参数解释：

    • --accept-eula：同意一些协议，需要
    • --name：容器的名称，方便后面管理容器

    使用

    sqlcmd query

    可以安装一个 azure data studio，然后在命令行中打开

    sqlcmd open ads

    如果不需要容器了，就可以删除

    sqlcmd delete

    如果想从其他工具连接，可以使用下面的命令找到连接信息

    sqlcmd config connection-strings

    后续使用

    启动容器

    docker start mssql

    开始编辑可以使用下面的命令，或者自己已经连接的其他工具

    sqlcmd open ads

    参考：

    • sqlcmd - 使用 sqlcmd 实用工具 - SQL Server | Microsoft Learn

    docker 安装 sqlserver

    安装

    拉取 sqlserver 镜像

    docker pull mcr.microsoft.com/mssql/server:2022-latest

    运行容器，将MSSQL_SA_PASSWORDS设置自己的密码

    docker run -e "ACCEPT_EULA=Y" -e "MSSQL_SA_PASSWORD=Kilos.123" -p 1433:1433 --name sql1 --hostname sql1 -d mcr.microsoft.com/mssql/server:2022-latest

    参数解释：

    • -e "ACCEPT_EULA=Y"：接受用户许可
    • -e "MSSQL_SA_PASSWORD=YourStrong@Passw0rd" 设置密码
    • -e "MSSQL_COLLATION=" 指定排序规则
    • -p 1433:1433 端口映射
    • --name sql1 容器名称
    • --hostname sql1 设置容器主机名称
    • -d 容器后台运行
    • mcr.microsoft.com/mssql/server:2022-latest 容器镜像

    运行的时候发现端口被占用，但是没有找到，解决方法是 WinNAT 服务重启一下，需要管理员权限

    net stop winnat // 停止WinNAT服务
net start winnat // 重新启动WinNAT服务

    安装一个 sqlcmd

    winget install sqlcmd

    修改系统管理员密码，我不修改了

    docker exec -it sql1 /opt/mssql-tools/bin/sqlcmd `
   -S localhost -U SA -P "<YourStrong@Passw0rd>" `
   -Q "ALTER LOGIN SA WITH PASSWORD='<YourNewStrong@Passw0rd>'"

    连接到 sqlserver

    docker exec -it sql1 "bash"

    在容器内部进行连接

    sudo /opt/mssql-tools/bin/sqlcmd -S localhost -U SA -P "Kilos.123"

    从外部进行连接

    sqlcmd -S localhost,1433 -U SA -P "Kilos.123"

    不使用的时候关掉容器

    docker stop sql1

    后续使用

    启动容器

    docker start sql1

    正常连接使用就可以了，后面不想要了就可以删除关闭

    docker stop sql1
docker rm sql1
docker rmi mcr.microsoft.com/mssql/server:2022-latest

    docker 网络代理

    设置代理来拉取镜像

    配置 /etc/docker/daemon.json

    {
  "proxies": {
    "http-proxy": "http://127.0.0.1:7890",
    "https-proxy": "http://127.0.0.1:7890",
    "no-proxy": "localhost,127.0.0.1"
  }
}

    容器内使用代理

    首先，使用 docker run 的时候需要添加 --net host，让容器使用宿主网络，也就是当前电脑的网络

    然后在命令行中使用代理命令：

    export https_proxy=http://127.0.0.1:7890 http_proxy=http://127.0.0.1:7890 all_proxy=http://127.0.0.1:7890

    查看是否配置成功

    docker run --net host --rm alpine sh -c "wget -qO - ifconfig.co/country"

    等效按键

    • Meta 键等效于Alt，如果是按住Alt按键，也等效于Esc，但是Esc 等效于 C-[
    • RET 等效于 C-m
    • TAB 等效于 C-i
    • C-_ 等效于 C-/
    • DEL 等效于 C-?，暂时没有测出来

    举个例子，如果想要按 M-RET 可以使用 C-[ C-m

    连接 github

    设置用户信息
